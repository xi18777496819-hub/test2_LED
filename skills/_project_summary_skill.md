# 项目总结 & 技能生成

> 项目完成后说"总结项目，生成技能"，Claude 按此模板执行。

## 我的编程偏好

1. **模块拆分** — 不同功能写在不同 .c/.h，main.c 只留业务逻辑
2. **中文注释** — 代码和文档用中文
3. **先简后繁** — 先跑通再优化
4. **分层设计** — gpio.c(硬件) → led.c(抽象) → main.c(业务)
5. **CubeMX 兼容** — 代码写在 USER CODE 标记之间

## 技能生成流程

当我说"总结项目"时，Claude 应：

1. 回顾对话中遇到的问题和解决方案
2. 提取可跨项目复用的经验（硬件参数、工具配置、排错流程）
3. 按以下模板生成技能文件：

```markdown
# <标题>
> 来源: <项目名> | <日期>

## 问题/场景
（什么情况下需要）

## 解决方案
（步骤、命令、代码）

## 踩过的坑
（失败的尝试和原因）
```

4. 文件放到 `.claude/skills/`，文件名用英文短横线命名

## 工具链

- **硬件**: STM32F103C8T6 Blue Pill
- **环境**: CubeMX + HAL + CMake + arm-none-eabi-gcc + VSCode
- **调试器**: 克隆版 ST-Link (SWD)，烧录用 `connectUnderReset` 模式
